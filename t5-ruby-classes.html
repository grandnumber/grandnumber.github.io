Classes & Cars ... without hitting the road

4th January 2014

Today we are looking at ruby classes - and we are going to construct a car class. You can think of the class as a way of creating our car and class methods as actions we want our car to do. First up - creating the car

1. Create the basic car

Constructing the car class is as simple as initiliazing it using the class approach. The code below allows us to create a new class Car and initializes a new instance variable @car. Instance variables can be referenced and accessed by class methods (actions that the class can take) :


  class Car
    def initialize
    @car = ""
    end
  end

2. Going vroom vroom...

So to have our car do anything we need to create some methods inside our class which will be accesses by any new car we create ... 



  class Car

  def initialize
    @car = ""
    @driver =""
  end

  def type
    puts "Please enter your car type here:"
    @car = gets.chomp
  end

  def driver
    puts "Please tell me what type of driver you are here:"
    @driver = gets.chomp
  end

  def vroom
    puts "My #{@car} likes to go, vroom vroooommm..."
  end

  def reliability
    puts "I drive a #{@car} and I am a #{@driver} driver!! "
  end

end

3. Watching our car in method action ...

Getting our car to do things is as simple as initializing a new car and calling the methods we created for it. See below for an example of a Toyota car driven by a terrible driver - clearly noone we know ... 


my_wheels = Car.new
my_wheels.type // Toyota
my_wheels.driver // terrible
my_wheels.vroom // My Toyota likes to go, vroom vroooommm...
my_wheels.reliability // I drive a Toyota and I am a terrible driver!
