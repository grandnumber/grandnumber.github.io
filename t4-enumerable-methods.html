Mapping your way around arrays... things hard to write catchy titles for ...

14th December 2014

Ruby has proven itself as truly intuitive and closer to english than any other language I have played with to date. And it is, within reason, very user friendly - one of the great parts of that is the Ruby Enumerables module. In very simple terms the 'Enumerable Module' provides tonnes of iteration methods. Iteration Methods?

1. Iteration Methods?!

Yep - iteration methods are simply ways in which to loop, pass or run through an object, array or other enumerable objects. What's that? Looping through an array - means taking each item in an array in turn, but not stopping til the end (or where you want to) - a very basic example below:

This is the Each method on an array used to print each item to the console in turn :

 array = [1,2,3,4,5]
              array.each |i| do
                puts i
              end
//:
              1
              2
              3
              4
              5

2. And so to map...

The map enumerable method is a powerful enumerable for arrays. Invokes the given block once for each element of self. Creates a new array containing the values returned by the block.

What that means in simple terms is that you can use the map function to easily change every item in an array with one line - for instance, you can double every value, halve it, add to it etc..

This is the Map method on our same array used to double each item & print the result to the console :

 puts array.map { |x| x*2 }
            //
              [2,4,6,8,10]

3. You can keep it light or go serious with the map method too !!

In the above example the map method returned you the result from running the map function. But it did not alter the original array. Which is handy when you want to try something and not permanently alter your sources etc. But sometimes you want to get serious with your changes - like serious! and intuitively Ruby allows you to do that by running the map! function instead of the map function. In simple terms: map will run your method and tell you the results but preserve your original array (a non-destructive method). In contrast the map! function will run your method against your array and then CHANGE that array to the result (a destructive method).

Example 1: The Map method doubles each item & returns the result, but when we print the array to the console it is preserved :

 puts array.map { |x| x*2 }
            //
              [2,4,6,8,10]
 puts array
            //
              [1,2,3,4,5]

Example 2: The Map! method doubles each item & returns the result, but when we print the array to the console it is preserved :

 puts array.map! { |x| x*2 }
            //
              [2,4,6,8,10]
 puts array
            //
              [2,4,6,8,10]
