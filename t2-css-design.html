CSS - a primer on the basics! why is all this specificity useful?

28th November 2014

I have been battling with CSS all week - gradually uncovering more and more about the rules of specificity and inheritance. So in the spirit of discovery...

1. Margins really ARE different to Borders & Padding:

The CSS box model is well known - essentially every element can be represented by a box. If we take a simple piece of text in its content box, we can then style it by adding some space around the text, we can then add a border around it and then add a margin around that.

One key thing to note is that margins are collapsible - padding and borders are not. When two elements are styled side by side they will be separated by the amount of both their padding ad both their borders BUT only the greater of both their margins. In this way we can ensure that the space between adjoining elements is equal to the margin we set. This makes intuitive sense if we think of old school essays where we want 1.5 spacing on our essays. Effectively each line needs a margin (spacing) of 1.5xlines. But if we add the 1.5x margin of the top line to the 1.5x margin of the second line we would end up with a 3x spacing - double what we wanted. So we collapse the margins - ultimately both lines want to achieve a 1.5x margin.

2. IDs beat classes, but some classes beat other classes too:

Classes and ID's are used to select specific html elements. They are very similar - both provide a logical shorthand to select our elements in CSS. However, while we can have as many elements of a certain class type as we want we can only have one element of a certain id type. Thus, Classes are good for commonly recurring elements such as buttons. ID's are good for major structural elements like headers or other unique elements like navbars.

Confusingly, an element can have both classes and an id specified, so it is necessary to understand which style will be applied. When the two conflict the ID will win as it has greater specificity and CSS prioritises it. In the event that two classes are applied to the same element and they conflict, then the first listed class (in the html assignment) will win.
